# Development Rules

Before creating files:

1. Check whether file already exists.
2. Reuse existing types.
3. Reuse existing components.
4. Prefer composition over duplication.
5. Follow AGENTS.md.
6. Follow SECURITY_RULES.md.
7. Follow PERFORMANCE_RULES.md.

When uncertain:
Present plan before implementation.

Never generate placeholder code unless requested.

Keep architecture consistent.

Prefer maintainability over speed.
